# Web年賀状（ローカル版）

ブラウザだけで宛名面・挨拶面を編集し、そのまま印刷まで完結できるシングルページアプリです。ビルド不要で `index.html` を静的サーバーで開くだけで動作します。

## 使い方
- 左下「編集」を押すとエディタが開きます。PCでは右側のドックパネル、モバイルでは下部シートとして開き、**カードを見ながら**編集できます。
- 宛名・敬称・差出人・挨拶文・背景を入力／選択すると即時プレビューに反映されます。編集中の項目に応じてカードの表裏が自動で切り替わり、カードをタップして手動で反転することもできます。
- エディタは「×」ボタンまたは Esc キーで閉じます。
- 右下の「印刷」ボタン（PCのみ）から印刷プレビューを開きます。モバイルはブラウザ標準の印刷から出力してください。
- カードはクリック / Enter / Space / 左右スワイプで裏表反転します。

## 主な機能
- 宛名面と挨拶面のリアルタイム編集・プレビュー
- 背景画像のサンプル適用・任意画像アップロード（Cropper.jsで100:148にトリミング）
- 挨拶文メッセージの自動フィット（PCでの改行位置をモバイルでも維持）
- 画像／メッセージ縦位置の微調整（ローカルに保存）
- CSV一括モードで宛名リストから共有URLを生成
- **年賀状ガチャ演出（ウマ娘パロディ・2027）**: 共有リンクで開いた閲覧者が「タップして裏面を見る」を押すと、突然ガチャ演出が始まり、SSR確定（★3）として年賀状の裏面が排出されるジョーク機能

## 年賀状ガチャ演出について
- 発動条件: `#id=…&data=…` 付きの共有リンクで開き、宛名面を初回タップしたとき（エディタ表示中・2回目以降は通常のフリップ）
- 流れ: 扇子「激熱ッ！」→ 赤い廊下を年賀ハガキが舞い馬蹄の扉へ → 扉開放 → ペンライトの観客席 → スタートゲート全景 → 枠番パン（1番=金 →10番=虹） → 発走ランプ点灯 → 虹ゲートから発走（ハガキ疾走） → セリフ → SSR確定演出（実データの裏面が名前帯・★3付きで登場） → タップで本物の裏面へ
- ハガキの表裏テクスチャは共有データ（宛名・差出人・挨拶文・背景URL）から Canvas で生成
- 無効化: URLに `?nogacha` を付ける（例: `…/?nogacha#id=…`）
- デバッグ: `?gphase=フェーズ名` で任意フェーズから自動再生（fan / dash / doorOpen / crowd / sky / whiteflash / gateWide / gatePass / lamp / goldGate / run / speech / reveal）
- 実装: `js/gacha/`（three.js + gsap を CDN の import map で解決、ビルド不要）と `gacha.css`。エンジンは共有リンク閲覧時のみアイドル時間に遅延ロード
- 本演出はウマ娘のガチャ演出のファンメイド・パロディです（Cygames とは無関係。画面内への注釈表示はなし）

## Google Analytics（カードID単位の計測）
`js/card-analytics.js` が共有リンクの `#id=…` と、data パラメータから復号した **宛名（recipient）・差出人（sender）** を全イベントに付与して GA4 に送ります。

送信イベント（すべて `card_id` / `recipient` / `sender` 付き）:
- `card_view` … 共有リンクが開かれた
- `gacha_start` / `gacha_skip`（`phase` 付き） / `gacha_complete`（`duration_sec` 付き） … ガチャ演出の開始/スキップ/完走
- `card_flip_to_back` … 裏面（挨拶面）に初めて到達
- あわせてユーザープロパティ `card_id` / `recipient` / `sender` も設定

GA4 側の設定（初回のみ・管理画面での作業）:
1. GA4 管理 → データの表示 → **カスタム定義** → カスタムディメンションを作成
   - `card_id` / 範囲: **イベント** / イベントパラメータ: `card_id`
   - `recipient` / 範囲: **イベント** / イベントパラメータ: `recipient`
   - `sender` / 範囲: **イベント** / イベントパラメータ: `sender`
2. 反映後、「探索」で ディメンション=recipient（または card_id）× 指標=イベント数、
   もしくはレポートのフィルタで `recipient = 宛名 太郎` のように指定すると、
   その相手の開封・ガチャ完走・裏面到達が個別に確認できます。
3. 登録前のイベントでも Realtime や BigQuery エクスポートでは各パラメータを参照できます。

※ 宛名・差出人は個人名です。GA の利用規約上 PII の送信は自己責任となる点に留意してください（自分専用のプロパティで、送る相手も把握している前提の機能です）。

## ファイル構成
- `index.html` : 画面本体とエディタUI
- `nenga.css` : レイアウト、3Dカード表現、印刷スタイル
- `js/`  
  - `main.js` : 初期化エントリ  
  - `card-controls.js` : 裏表反転・操作入力  
  - `card-effects.js` : パララックス・光沢・ジャイロ  
  - `editor.js` : 入力反映・共有URL生成・背景アップロード・CSVモード  
  - `position-controls.js` : 画像/挨拶文の縦位置調整  
  - `message-autofit-lite.js` : 挨拶文の自動フィット（軽量版）  
  - `print.js` : 画面を画像化して印刷（宛名縦積みなどの前処理を含む）  
  - `image-utils.js`, `storage.js`, `templates.js`, `utils.js`, `csv-mode.js` ほか補助スクリプト

## 開発メモ
- ビルドレス構成です。編集後はブラウザをリロードするだけで反映されます。
- 共有URL機能や外部ストレージ連携は`storage.js`などに設定があります（必要に応じて差し替えてください）。
- 印刷はhtml2canvasでDOMを画像化して2ページ（挨拶面/宛名面）を生成しています。

## 印刷のポイント
- PC：右下の「印刷」ボタンを使用。ブラウザの印刷設定で「背景のグラフィック」を有効にしてください。
- モバイル：ブラウザ標準の印刷機能から出力してください（表示と同じ幅でキャプチャします）。
- 宛名面は横書きテキストを`<br>`で縦積みし、スペースは空行として扱います。
